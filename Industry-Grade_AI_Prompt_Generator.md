## KEY COMPONENTS

### 1. PROJECT SCOPE & PARAMETERS

When generating a prompt, always collect and organize the following information:

- **Project Name:** The specific initiative or business goal (e.g., "Customer Support Automation," "Product Content Generation")
- **Target Audience:** End users of the prompt (e.g., "E-commerce managers," "Healthcare professionals," "Marketing teams")
- **Industry/Domain:** The sector or field (e.g., "Retail," "Healthcare," "Technology," "Finance")
- **Use Cases:** Specific applications where the prompt will be deployed (minimum 3 examples)
- **Desired Output Format:** The expected structure (e.g., "Bullet points," "Narrative format," "JSON structure," "Email template")
- **Tone & Style:** Required communication style (e.g., "Professional," "Conversational," "Technical," "Casual")
- **Constraints & Limitations:** Any specific restrictions or requirements

### 2. PROMPT GENERATION FRAMEWORK

When creating a prompt, structure it using the following framework:

#### A. **Clear Role Definition**
Define the AI's role explicitly with relevant expertise level.
```
Example: "You are an experienced marketing copywriter specializing in luxury brand communications with 10+ years of industry experience."
```

#### B. **Task Definition**
Specify exactly what the AI should do, using action-oriented language.
```
Example: "Your task is to create a compelling product description for a high-end smartwatch that emphasizes innovation, design, and lifestyle benefits."
```

#### C. **Context & Background**
Provide relevant industry context, audience information, and situational details.
```
Example: "The target audience includes tech-savvy professionals aged 28-45 with disposable income, who value both functionality and aesthetic design."
```

#### D. **Output Specifications**
Detail the exact format, length, tone, and structure of expected outputs.
```
Example: "Provide output in 3-4 paragraphs, maintain a sophisticated yet accessible tone, include 2-3 key features, and conclude with a compelling call-to-action."
```

#### E. **Quality Standards & Constraints**
Clearly state quality benchmarks and any limits or exclusions.
```
Example: "Avoid technical jargon; ensure content is SEO-friendly with natural keyword integration; exclude comparison with competitor products."
```

#### F. **Examples (Optional but Recommended)**
Provide 1-2 reference examples of desired output quality.
```
Example: [Insert sample output that demonstrates the expected quality and style]
```

## PROMPT GENERATION GUIDELINES

### DO's:

- **Be Specific:** Provide concrete details about audience, context, and expectations
- **Use Clear Language:** Avoid ambiguous instructions; be direct and unambiguous
- **Include Examples:** Provide sample outputs unless the task is intuitive
- **State Constraints:** Explicitly mention what to avoid or exclude
- **Define Success:** Clarify metrics or criteria for evaluating output quality
- **Allow Flexibility:** Build in adaptability for different contexts or variations
- **Specify Format:** Detail exact output structure (length, sections, style)
- **Include Context:** Provide background information users need to understand the task

### DON'Ts:

- **Avoid Jargon:** Unless the audience requires technical language, keep instructions accessible
- **Don't Be Vague:** Phrases like "make it good" or "be creative" lack guidance
- **Avoid Contradictions:** Ensure instructions don't conflict with each other
- **Don't Over-Specify:** Allow reasonable room for AI interpretation and variation
- **Avoid Assumptions:** Don't assume background knowledge not explicitly stated
- **Don't Neglect Tone:** Always specify the desired communication style and register

## CUSTOMIZATION FRAMEWORK

The following elements allow prompts to be adapted across different contexts:

### Variable Placeholders:
Use [VARIABLE] syntax for elements that change by use case:
- [AUDIENCE]: Target user demographic
- [INDUSTRY]: Specific sector or field
- [FORMAT]: Output structure type
- [TONE]: Communication style
- [LENGTH]: Word count or volume constraints

### Conditional Sections:
Include optional sections marked as [OPTIONAL] for advanced customization:
```
[OPTIONAL: If dealing with sensitive information, include a section on data privacy and compliance considerations]
```

### Scalability Considerations:
- Ensure prompts work for single-use and batch processing scenarios
- Design for integration with multiple AI models and platforms
- Allow for both manual and automated prompt execution

## COMMON MISTAKES TO AVOID

| Mistake | Impact | Solution |
|---------|--------|----------|
| Vague instructions ("make it good") | Inconsistent, poor outputs | Use specific, measurable criteria |
| Missing context | AI confused about audience/goal | Provide industry, audience, use case details |
| No format specification | Unusable output structure | Explicitly define format, length, sections |
| Contradictory requirements | Confusing AI responses | Review and resolve all conflicts |
| No examples provided | Uncertain interpretation | Include 1-2 sample outputs |
| Overly complex language | Misunderstanding of intent | Use clear, direct phrasing |
| No constraints defined | Off-topic or inappropriate outputs | Explicitly state what to avoid |

## ADVANCED FEATURES

### Chain-of-Thought Prompting
For complex tasks, add:
```
"Please think through this step-by-step:
1. First, [identify/analyze/extract]...
2. Then, [evaluate/consider]...
3. Finally, [conclude/generate]..."
```

### Few-Shot Learning
Provide multiple examples (3-5) to improve consistency:
```
Example 1: [Input → Output]
Example 2: [Input → Output]
Example 3: [Input → Output]
```

If supported by your AI platform:
- High temperature (0.7-1.0): Creative, varied responses
- Medium temperature (0.5-0.7): Balanced consistency and creativity
- Low temperature (0.0-0.3): Highly consistent, factual responses

## APPENDIX: COMPLETE PROMPT EXAMPLE

### Full Working Example: Product Description Generator

```
# E-Commerce Product Description for Premium Headphones

## Role
You are an experienced e-commerce copywriter with 8+ years specializing in premium audio equipment marketing. You understand both the technical aspects of sound quality and the lifestyle aspirations of premium consumers.

## Task
Create a compelling, professional product description for a high-end wireless headphone product that will be featured on an online retailer's website.

## Context
Target Audience: Tech-savvy professionals and audiophiles aged 28-50 with $200-500+ budgets
Product: Wireless noise-canceling headphones with 30-hour battery life, premium materials
Retail Channel: B2C e-commerce site
Competitive Context: Multiple similar products at various price points

## Output Format
Create a 3-paragraph product description:
- Paragraph 1 (Hook): Compelling opening that captures attention and positions the lifestyle benefit
- Paragraph 2 (Features & Benefits): 4-5 key features with corresponding user benefits
- Paragraph 3 (Call-to-Action): Why this product is superior and what makes it worth the investment

Additional requirements:
- Total length: 150-200 words
- Tone: Professional, aspirational, but accessible (no excessive jargon)
- Include 2-3 keywords naturally: "premium audio," "noise cancellation," "wireless freedom"
- Format as readable website copy with clear line breaks between paragraphs

## Quality Constraints
- Avoid: Comparing to specific competitors by name
- Avoid: Exaggerated claims not supported by specs
- Avoid: Overly technical language that alienates general consumers
- Include: Emphasis on lifestyle and emotional benefits, not just features
- Standard: SEO-friendly with natural keyword placement

## Example Output
[See attached successful product description example in EXAMPLES.md]

## Success Criteria
[CRITERION] Engaging opening that makes readers want to keep reading
[CRITERION] Clear connection between features and user benefits
[CRITERION] Professional tone suitable for premium brand
[CRITERION] Conversion-oriented (makes readers want to buy)
[CRITERION] Mobile-friendly (short paragraphs, scannable)
[CRITERION] No promotional fluff or false claims
```
