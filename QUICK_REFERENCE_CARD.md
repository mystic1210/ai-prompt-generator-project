## THE 6 ESSENTIAL COMPONENTS

Every industry-grade prompt needs these 6 sections:

### 1. ROLE
Define who the AI is and their expertise level.
```
You are a [profession/title] with [X years] of experience in [field].
```
Example: "You are a marketing copywriter with 10+ years of experience in luxury brands."

---

### 2. TASK
State exactly what the AI should do in action-oriented language.
```
Your task is to [specific action].
```
Example: "Your task is to create a product description that emphasizes luxury and exclusivity."

---

### 3. CONTEXT
Provide relevant background about audience, industry, and situation.
```
Context:
- Audience: [demographic and psychographic details]
- Industry: [specific sector]
- Situation: [relevant background]
```
Example: "Target audience: wealthy professionals aged 35-55 who value quality and status."

---

### 4. OUTPUT
Specify the exact format, length, tone, and structure.
```
Format: [type of output]
Length: [word count or volume]
Tone: [professional/casual/creative/etc.]
Structure: [specific sections or organization]
```
Example: "3 paragraphs (150-200 words), professional tone, benefit-focused language."

---

### 5. CONSTRAINTS
Clearly state what to do and what to avoid.
```
DO:
- [Requirement 1]
- [Requirement 2]

DON'T:
- [What to avoid 1]
- [What to avoid 2]
```
Example: "DO include sensory language. DON'T compare to competitors by name."

---

### 6. EXAMPLES
Provide 1-2 sample inputs and expected outputs (optional for simple tasks).
```
Input: [sample question/content]
Output: [desired response]
```

## VALIDATION CHECKLIST (Quick Version)

Before using any prompt, verify:

- [ ] Role is clearly defined with expertise level
- [ ] Task is specific, unambiguous, and actionable
- [ ] Context provides necessary background
- [ ] Output format is explicitly detailed (format, length, tone)
- [ ] DO/DON'T constraints are clear and specific
- [ ] Example provided (if needed for complex tasks)
- [ ] Language is professional and accessible
- [ ] No contradictions or conflicting instructions
- [ ] Suitable for your industry/application
- [ ] Aligned with compliance requirements (GDPR, HIPAA, etc.)

**[READY] = Ready to use | [NEEDS WORK] = Needs improvement**

## QUICK TIPS

### DO's
- Be specific and precise
- Use clear action verbs (create, write, analyze, generate)
- Provide concrete examples
- State constraints explicitly
- Mention success criteria
- Keep language professional but accessible

### DON'Ts
- Use vague instructions ("make it good")
- Assume background knowledge not stated
- Create contradictory requirements
- Skip format specifications
- Use overly technical jargon
- Leave room for misinterpretation

## COMMON PROMPTS (Templates)

### Product Description
"You are a [role]. Create a [type] product description for [product] targeting [audience]. Focus on [key benefit]. Format: [structure]. Constraints: [DO/DON'T]."

### Email Response
"You are a [customer service expert]. Write a professional response to this customer concern: [issue]. Maintain [tone]. Address [specific need]."

### Social Media Post
"You are a [social media expert]. Create a [platform]-optimized post for [audience] about [topic]. Goal: [engagement/clicks/shares]. Include [CTA type]."

### Technical Documentation
"You are a [technical writer]. Create documentation for [system/feature] for [skill level] users. Include: [prerequisites/steps/examples]. Avoid [technical jargon level]."

### Blog Post/Article
"You are a [subject matter expert]. Write a [length] blog post about [topic] for [audience]. Format: [structure]. Tone: [style]. Include [specific elements]."

## RED FLAGS (Fix These!)

| Issue | Fix |
|---------|-----|
| "Make it good" / "Be creative" | Replace with specific criteria |
| No length specification | Add word count or volume |
| No tone/style guidance | Specify professional/casual/technical/etc. |
| Missing audience details | Describe who will use this |
| Contradictory requirements | Review and resolve conflicts |
| No format specification | Define structure (paragraphs/bullets/sections) |
| No examples | Add 1-2 sample inputs/outputs |

- Emphasize: Empathy, solutions, relationship
- Constraints: No generic templates, personalize

## ADVANCED TECHNIQUES

Once you master basics:

- **Chain-of-Thought:** Ask AI to "think through step-by-step"
- **Few-Shot Learning:** Provide 3-5 examples instead of 1
- **Role Stacking:** Combine multiple roles for complex tasks
- **Constraint Layering:** Add conditional constraints
- **Dynamic Variables:** Use [VARIABLE] placeholders for customization

## QUICK TROUBLESHOOTING

| Problem | Solution |
|---------|----------|
| Outputs too generic | Add more specific context |
| Outputs inconsistent | Make instructions clearer |
| Outputs off-topic | Add explicit constraints |
| Outputs too brief | Specify longer length |
| Outputs too technical | Add "avoid jargon" |
| Outputs have errors | Add fact-checking requirement |

1. **Be Specific** → Replace vague phrases with concrete details

3. **State Constraints** → Explicit limits prevent bad outputs
4. **Show Examples** → Sample outputs clarify expectations
5. **Validate Thoroughly** → Check before deploying
6. **Test & Iterate** → Real outputs reveal improvements
